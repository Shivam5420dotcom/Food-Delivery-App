1. Introduction

With the increasing use of digital platforms, online food delivery systems have become an essential part of everyday life. Applications like Swiggy, Zomato, and Uber Eats have set a benchmark in providing quick and convenient food ordering solutions.

This project, FoodYum, is a simplified version of an online food ordering system designed using HTML and CSS. It focuses on the core idea of enabling users to browse different food categories and navigate to detailed pages of individual items.

2. Project Description

FoodYum – Online Food Ordering System is a static website built using HTML and CSS that demonstrates the structure and flow of a modern food delivery platform.

The website includes:

  • A homepage (food.html) displaying brand name, search bar, login button, and multiple food category cards such as Breakfast, Lunch, Dinner, Beverages and Desserts.
  • Each category card redirects the user to its individual page (e.g., breakfast.html, lunch.html, etc.).
  • Each category page contains multiple food item cards with images and titles.
  • A neat and user-friendly interface using CSS, inspired by real food-ordering applications.

The goal of FoodYum is to represent how users browse, view categories, and navigate to specific food sections, forming the base of a full food ordering UI.

3. Objective

The primary objectives of this project are:

  • To understand and apply basic web development structure using HTML and CSS.
  • To create a clean, responsive, and user-friendly UI for a food ordering website.
  • To learn about navigation between multiple web pages.
  • To understand folder structuring for web projects.
  • To follow the concept of component-based design such as cards, category sections, and layout elements.

4. Scope of the Project

The project covers the front-end interface of an online food ordering system. It shows:

  • Homepage UI
  • Category cards
  • Food item cards
  • Page navigation
  • Visual design structure

Since the project is static, it does not include backend features like payments or databases, but it acts as a strong foundation for building a complete web application in the future.

5. Technologies Used

  • HTML5 — for creating the structure of the website.
  • CSS3 — for styling, layout design, card UI, fonts, shadows, alignment, etc.
